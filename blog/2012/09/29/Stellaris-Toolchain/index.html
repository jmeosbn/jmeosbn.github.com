Setting Up the Stellaris Toolchain

Sep 29th, 2012

Mostly you can follow the instructions at y3xz, but there were a couple of catches on my MacBook as detailed below.

toolchain

The makefile failed to download the source archive from mentor, you can download it manually into the root of the toolchain repo and make will continue so long as the checksum matches.

lm4tools

The lm4tools binary couldn’t read the serial number of the device using my MacBook running OS X 10.8 so the binary wasn’t flashed.

The simplest workaround is to checkout and build commit cc49426, which doesn’t check for the serial number. To use newer versions of lm4tools, a kernel extension needs to be installed. For more info, see the issue on github.

verify toolchain

# Add the toolchain to your path
$ export PATH=$HOME/arm-cs-tools/bin:$PATH

# Test the compiler
$ cd ../stellaris/boards/ek-lm4f120xl/project0/
$ make clean; make

# Flash binary to the board
$ lm4flash gcc/project0.bin

# Try making a source modification
$ nano project0.c
$ make && lm4flash gcc/project0.bin

debugging

# build with debug symbols
$ make clean && DEBUG=1 make
$ lm4flash gcc/project0.bin

# start tcp/usb bridge in a separate terminal
$ lmicdi

# start gdb and connect to device
$ arm-none-eabi-gdb gcc/project0.axf
Reading symbols from ./gcc/project0.axf...done.
(gdb) target remote :7777
